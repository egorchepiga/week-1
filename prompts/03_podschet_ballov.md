# Этап 3: Подсчёт баллов для гипотез

## Цель этапа
Объективно сравнить все найденные идеи и выбрать лучшую для запуска.

## Система оценки

### Критерии и веса

| Критерий | Вес | Описание |
|----------|-----|----------|
| Потенциал монетизации | x2 | Возможность заработка (из этапа 2) |
| Объём трафика | x2 | Количество поисковых запросов |
| Конкуренция | x1.5 | Насколько занята ниша |
| Простота реализации | x1 | Сложность разработки |
| Понятность продукта | x1 | Легкость объяснения пользователю |

### Шкала оценки (1-5)

```
5 - Отлично, идеальные условия
4 - Хорошо, есть небольшие минусы
3 - Средне, типичная ситуация
2 - Плохо, есть серьёзные проблемы
1 - Очень плохо, критические недостатки
```

## Алгоритм действий

### Шаг 3.1: Создание матрицы оценки
```
ТАБЛИЦА:
| Идея | Монетизация (x2) | Трафик (x2) | Конкуренция (x1.5) | Простота | Понятность | ИТОГО |
|------|------------------|-------------|---------------------|----------|------------|-------|
| #1   |                  |             |                     |          |            |       |
| #2   |                  |             |                     |          |            |       |
| ...  |                  |             |                     |          |            |       |
```

### Шаг 3.2: Оценка каждой идеи
```
ДЛЯ КАЖДОЙ ИДЕИ:
  1. Оценить монетизацию (данные из этапа 2)
  2. Оценить трафик (данные из Semrush, этапы 4-5)
  3. Оценить конкуренцию (проверка занятости запросов)
  4. Оценить простоту (можно ли сделать за 1-2 недели?)
  5. Оценить понятность (объясните за 10 секунд)
```

### Шаг 3.3: Расчёт итогового балла
```
ИТОГО = (Монетизация × 2) + (Трафик × 2) + (Конкуренция × 1.5) + Простота + Понятность

МАКСИМУМ = (5×2) + (5×2) + (5×1.5) + 5 + 5 = 37.5 баллов
МИНИМУМ = (1×2) + (1×2) + (1×1.5) + 1 + 1 = 7.5 баллов
```

### Шаг 3.4: Принятие решения
```
ПРАВИЛО: Запускаем идею с максимальным количеством баллов

ИСКЛЮЧЕНИЯ:
  - Если топ-2 идеи близки по баллам (разница <3) — выбираем более простую
  - Если все идеи <20 баллов — ищем новые идеи
```

## Детализация критериев

### Монетизация (1-5)
```
5: Подписка $10+/мес, B2B, доказанная готовность платить
4: Подписка $5-10/мес, профессиональные инструменты
3: Подписка $3-5/мес, массовый рынок
2: Только продажа, нет подписок у конкурентов
1: Бесплатные аналоги доминируют
```

### Трафик (1-5)
```
5: >10K запросов/мес в US (без хвостов)
4: 5-10K запросов/мес в US
3: 2-5K запросов/мес в US
2: 500-2K запросов/мес в US
1: <500 запросов/мес в US
```

### Конкуренция (1-5)
```
5: Запрос свободен, нет оптимизированных конкурентов
4: 1-2 слабых конкурента
3: Есть конкуренты, но можно отстроиться
2: Сильные конкуренты, но есть ниша
1: Запрос полностью занят топ-расширениями
```

### Простота реализации (1-5)
```
5: 1 простая функция, есть готовый код на GitHub
4: 1 функция, нужно немного допилить
3: 2-3 функции, стандартная разработка
2: Сложная логика, интеграции с API
1: Требует бэкенда, ML, сложных вычислений
```

### Понятность продукта (1-5)
```
5: Понятно из названия за 3 секунды
4: Понятно из описания за 10 секунд
3: Нужна демонстрация
2: Требует обучения
1: Сложно объяснить даже за минуту
```

- [ ] Выбрана идея с максимальным баллом
- [ ] Записано обоснование выбора

## Пример заполненной матрицы
