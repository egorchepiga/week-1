# Этап 10: Вытаскиваем функцию и разрабатываем

## Цель этапа
Найти готовый код или создать MVP расширения с одной ключевой функцией.

## Главный принцип

```
ПРАВИЛО: Продукт должен сводиться к 1 ПРОСТОЙ функции

ПОЧЕМУ:
  ✓ Быстрее разработка (1-2 недели вместо месяцев)
  ✓ Проще объяснить пользователю
  ✓ Меньше багов
  ✓ Быстрее валидация идеи
  ✓ Легче итерировать

ВОПРОС ДЛЯ ПРОВЕРКИ:
  "Можно ли объяснить что делает расширение за 5 секунд?"
  Если НЕТ → слишком сложно
```

## Алгоритм поиска готового кода

### Шаг 10.1: Поиск на GitHub
```
ДЕЙСТВИЕ: Искать в Google по формату:
  [суть функции] chrome github

ПРИМЕРЫ:
  "track email chrome github"
  "block website chrome github"
  "screenshot page chrome github"
  "dark mode chrome github"

ЧТО ИСКАТЬ:
  ✓ Репозитории с открытым кодом
  ✓ Расширения с MIT/Apache лицензией
  ✓ Проекты с понятной структурой
```

### Шаг 10.2: Анализ найденного кода
```
ПРОВЕРИТЬ:
  1. Лицензия позволяет использование?
  2. Код не обфусцирован (читаемый)?
  3. Есть документация?
  4. Последнее обновление не древнее?
  5. Работает ли демо/пример?
```

### Шаг 10.3: Оценка сложности реализации
```
ЕСЛИ нашли готовый код:
  → Оценить время на адаптацию (обычно 1-3 дня)
  → Проверить совместимость с Manifest V3

ЕСЛИ НЕ нашли готовый код:
  → Оценить сложность с нуля
  → Рассмотреть альтернативные идеи
```

## Критерии оценки сложности

### Простая реализация (1-3 дня)
```
ПРИМЕРЫ:
  ✓ Изменение внешнего вида страницы (CSS injection)
  ✓ Блокировка элементов (по селектору)
  ✓ Копирование/форматирование текста
  ✓ Простые калькуляторы
  ✓ Таймеры и напоминания

ХАРАКТЕРИСТИКИ:
  - Работает только на клиенте
  - Не требует бэкенда
  - Использует стандартные Chrome APIs
```

### Средняя реализация (1-2 недели)
```
ПРИМЕРЫ:
  ✓ Интеграция с популярными API (Gmail, LinkedIn)
  ✓ Сохранение данных в storage
  ✓ Popup с настройками
  ✓ Content scripts с логикой

ХАРАКТЕРИСТИКИ:
  - Несколько компонентов
  - Требует отладки
  - Может требовать oauth
```

### Сложная реализация (>2 недель)
```
ПРИМЕРЫ:
  ✗ AI/ML функции
  ✗ Собственный бэкенд
  ✗ Синхронизация между устройствами
  ✗ Обработка больших данных

РЕКОМЕНДАЦИЯ: Избегать на старте!
```

## Вопросы для принятия решения

```
ВОПРОС 1: Можно ли сократить продукт до 1 функции?
  ДА → продолжаем
  НЕТ → ищем другую идею

ВОПРОС 2: Насколько проста реализация?
  ПРОСТАЯ → идеально для старта
  СРЕДНЯЯ → приемлемо с опытом
  СЛОЖНАЯ → рассмотреть другую идею

ВОПРОС 3: Есть ли готовый код на GitHub?
  ДА → ускоряет разработку в 3-5 раз
  НЕТ → оценить время на разработку с нуля
```

## Структура Chrome-расширения (Manifest V3)

```
my-extension/
├── manifest.json        # Конфигурация расширения
├── popup/
│   ├── popup.html       # UI popup окна
│   ├── popup.css        # Стили popup
│   └── popup.js         # Логика popup
├── content/
│   └── content.js       # Скрипт для страниц
├── background/
│   └── service-worker.js # Фоновый процесс
├── icons/
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── _locales/            # Переводы (для SEO!)
    ├── en/
    │   └── messages.json
    ├── ru/
    │   └── messages.json
    └── ... (30+ языков)
```

## Минимальный manifest.json

```json
{
  "manifest_version": 3,
  "name": "__MSG_extensionName__",
  "description": "__MSG_extensionDescription__",
  "version": "1.0.0",
  "default_locale": "en",
  "icons": {
    "16": "icons/icon16.png",
    "48": "icons/icon48.png",
    "128": "icons/icon128.png"
  },
  "action": {
    "default_popup": "popup/popup.html",
    "default_icon": "icons/icon48.png"
  },
  "permissions": ["storage"],
  "content_scripts": [{
    "matches": ["<all_urls>"],
    "js": ["content/content.js"]
  }]
}
```

## Подсказки для промптов

### Для поиска готового кода:
```
Найди open-source Chrome расширения на GitHub для функции:
[ОПИСАНИЕ ФУНКЦИИ]

Требования:
- Manifest V3 совместимость
- MIT или Apache лицензия
- Активный репозиторий
- Понятная структура кода
```

### Для генерации кода:
```
Создай минимальное Chrome расширение (Manifest V3) с функцией:
[ОПИСАНИЕ ФУНКЦИИ]

Требования:
- Только необходимый код
- Комментарии на русском
- Готово к публикации в Chrome Web Store
- Структура для локализации
```

### Для оптимизации существующего кода:
```
Оптимизируй код Chrome расширения:
[КОД]

Задачи:
1. Упростить где возможно
2. Убрать лишние зависимости
3. Добавить обработку ошибок
4. Подготовить к Manifest V3
```

### Для создания описания:
```
Напиши описание для Chrome Web Store (3000+ символов):

Расширение: [НАЗВАНИЕ]
Функция: [ОПИСАНИЕ]
Целевая аудитория: [АУДИТОРИЯ]
Ключевые слова: [КЛЮЧИ]

Требования:
- SEO-оптимизированное
- Включает ключевые слова естественно
- Описывает преимущества
- Call-to-action в конце
```

## Типичные ошибки

1. **Слишком много функций** — начинайте с 1
2. **Игнорирование Manifest V3** — старые расширения не примут
3. **Нет локализации** — теряете 90% трафика
4. **Короткое описание** — нужно 3000+ символов
5. **Плохие скриншоты** — влияют на конверсию
