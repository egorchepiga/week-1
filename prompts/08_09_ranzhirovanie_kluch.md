# Этапы 8-9: Ранжирование по запросам и проверка ключа

## Цель этапа
Оценить дополнительный трафик с хвостовых запросов и убедиться, что ключ "софтовый".

---

## ЭТАП 8: Ранжирование по другим запросам

### Зачем проверять хвосты

```
ХВОСТЫ (Long Tails) — вариации основного запроса

ПРИМЕР для "site blocker":
  - best site blocker
  - site blocker chrome
  - free site blocker
  - site blocker for chrome
  - how to block sites

ВАЖНОСТЬ: Хвосты могут дать 50-80% общего трафика!
```

### Алгоритм проверки в Keyword Magic Tool

```
ДЕЙСТВИЕ:
  1. Открыть Keyword Magic Tool в Semrush
  2. Ввести основной ключ
  3. Смотреть список запросов с вариациями

ЧТО АНАЛИЗИРОВАТЬ:
  ✓ Общий объём всех хвостов
  ✓ Какой % хвостов релевантен нашему продукту
  ✓ Есть ли "шум" (нерелевантные запросы)
```

### Оценка релевантности хвостов

```
ХОРОШО — большинство хвостов про наш софт:
  "site blocker" → 90% хвостов про блокировку сайтов
  ✓ Можем ранжироваться по многим запросам

ПЛОХО — много шума в хвостах:
  "tracker" → хвосты про фитнес-трекеры, GPS-трекеры, и т.д.
  ✗ Будем получать нерелевантный трафик
```

### Пример анализа

```
КЛЮЧ: "site blocker"

ХВОСТЫ В KEYWORD MAGIC TOOL:
┌──────────────────────────────┬─────────────┐
│ Запрос                       │ Объём (US)  │
├──────────────────────────────┼─────────────┤
│ site blocker chrome          │ 1,200       │ ← релевантно
│ best site blocker            │ 800         │ ← релевантно
│ free site blocker            │ 600         │ ← релевантно
│ block website chrome         │ 500         │ ← релевантно
│ site blocker extension       │ 400         │ ← релевантно
└──────────────────────────────┴─────────────┘

ВЫВОД: 100% хвостов релевантны → отличный ключ!
```

## ЭТАП 9: Проверка "софтовости" ключа

### Что такое "софтовый" ключ

```
СОФТОВЫЙ КЛЮЧ — запрос, ассоциирующийся с программами

СОФТОВАЯ ВЫДАЧА содержит:
  ✓ Сайты-сервисы
  ✓ Мобильные приложения
  ✓ Плагины и расширения
  ✓ Подборки приложений
  ✓ Другой софт

ШУМНАЯ (НЕ-софтовая) ВЫДАЧА содержит:
  ✗ Статьи и блоги
  ✗ Wikipedia
  ✗ Amazon, eBay (физические товары)
  ✗ Каталоги и объявления
  ✗ Услуги
```

### Алгоритм проверки

```
ДЕЙСТВИЕ: Вбить ключ в Google (без "chrome")

ПРИМЕР: Ищем "site blocker" (не "site blocker chrome")

АНАЛИЗ ВЫДАЧИ:
  1. Подсчитать типы результатов в топ-10
  2. Определить % софтовых результатов

ПРАВИЛО:
  ✓ >50% софтовых результатов → ключ подходит
  ✗ <50% софтовых результатов → ключ шумный
```

### Примеры хороших и плохих ключей

```
ХОРОШИЙ КЛЮЧ: "site blocker"
  Топ-10 Google:
  1. BlockSite (расширение) ← софт
  2. StayFocusd (расширение) ← софт
  3. Freedom (приложение) ← софт
  4. Cold Turkey (сервис) ← софт
  5. Подборка лучших блокировщиков ← софт
  ...
  Результат: 90% софт → ОТЛИЧНО

ПЛОХОЙ КЛЮЧ: "tracker"
  Топ-10 Google:
  1. Fitbit (фитнес) ← НЕ наш софт
  2. GPS трекеры (товары) ← НЕ софт
  3. DHL tracking ← НЕ наш софт
  4. Package tracker ← другая ниша
  ...
  Результат: <30% релевантный софт → ПЛОХО
```

### Что делать с шумным ключом

```
СТРАТЕГИИ:

1. Добавить уточняющее слово:
   "tracker" → "email tracker"
   "tracker" → "price tracker"

2. Сузить нишу:
   "checker" → "grammar checker"
   "checker" → "email checker"

3. Найти альтернативную формулировку:
   "blocker" (шумный) → "focus mode" (чистый)
```

## Подсказки для промптов

### Для анализа хвостов:
```
Проанализируй long-tail запросы для ключа "[КЛЮЧ]":
1. Какие основные вариации существуют?
2. Какой % вариаций релевантен для [ТИП ПРОДУКТА]?
3. Какой дополнительный трафик дают хвосты?
4. Есть ли "шумные" хвосты?
```

### Для проверки софтовости:
```
Оцени "софтовость" ключа "[КЛЮЧ]" для Chrome-расширения:
1. Какие типы результатов в топ-10 Google?
2. Какой % связан с программами/сервисами?
3. Есть ли шум (нерелевантные результаты)?
4. Подходит ли ключ для расширения?
```

### Для финального решения:
```
Прими решение по ключу "[КЛЮЧ]" для расширения:

Данные:
- Объём US (без хвостов): [X]
- Объём US (с хвостами): [Y]
- % релевантных хвостов: [Z]%
- % софтовой выдачи: [W]%
- KD: [N]%

Рекомендуй: брать / не брать / модифицировать
```

---

## Матрица принятия решений

| Критерий | Отлично | Хорошо | Плохо | Очень плохо |
|----------|---------|--------|-------|-------------|
| % релевантных хвостов | >80% | 60-80% | 40-60% | <40% |
| % софтовой выдачи | >70% | 50-70% | 30-50% | <30% |
| Финальная рекомендация | Брать | Брать с оптимизацией | Модифицировать | Искать другой |

---

## Итоговая формула оценки ключа

```
КЛЮЧ ПОДХОДИТ, ЕСЛИ:

✓ Объём US (без хвостов) ≥ 500 (узкая ниша) или ≥ 2000 (широкая)
✓ KD ≤ 70% или выдача уже содержит расширения
✓ Ключ не занят оптимизированными конкурентами
✓ >60% хвостов релевантны
✓ >50% выдачи Google = софт
✓ Название уникально

ЕСЛИ ХОТЯ БЫ 1 КРИТЕРИЙ НЕ ВЫПОЛНЕН:
  → Модифицировать ключ или искать альтернативу
```
