## Критерии оценки (0-10 баллов каждый)

### 1. Количество пользователей
**Балл: [ ] / 10**

- Текущее количество: [число]
- Рекомендация: от 10K пользователей
- Не гнаться за миллионами от известных брендов (Cisco, Grammarly, Avast, Adobe, Zoom)

### 2. Информация о заработке
**Балл: [ ] / 10**

- Встроена ли подписка? [ДА/НЕТ]
- Стоимость подписки: [$]
- Другая информация о доходах: [любые данные]
- Продукты с подпиской зарабатывают в десятки раз больше

### 3. Можно ли сократить до 1 простой функции?
**[ДА / НЕТ]**

> RED FLAG: Если НЕТ - отказываемся от идеи и ищем другую
> Можно брать второстепенную функцию, не обязательно основную
> Если продукт уже прост - можно сделать такой же с другим ключом

### 4. Простота реализации
**Балл: [ ] / 10**

- Есть ли open source аналог? [ДА/НЕТ]
- Ссылка на GitHub: [URL]
- Поиск: `[суть функции] chrome github`
- Продукты с открытым кодом в 100 раз проще в разработке

### 5. Насколько хорошо ищут ключ для названия?
**Балл: [ ] / 10**

- Найденный запрос: [ключевое слово]
- Volume (Semrush): [число] запросов/мес
- Цель: максимум трафика при минимуме конкуренции

### 6. Выражена ли софтовость у ключа?
**[ДА / НЕТ]**

> RED FLAG: Если в Google выдаче >50% НЕ софт (статьи) - отказываемся
> Хорошо: сайты-сервисы, мобильные приложения, плагины, софт

### 7. Занят ли ключ оптимизированным расширением?
**[ДА / НЕТ]**

> RED FLAG: Если ЗАНЯТ хорошо оптимизированным - ищем другой ключ
> Признаки оптимизации: длинное описание (>3000), много переводов (>30)

### 8. Keyword Difficulty (Semrush)
**Балл: [ ] / 10**

| Зона | Баллы |
|------|-------|
| Зелёная/Жёлтая | 10 |
| Оранжевая | 7-8 |
| Красная | 6 |
| Алая | 5 |

- KD значение: [ ]%
- Цвет зоны: [зелёная/жёлтая/оранжевая/красная/алая]

## Советы

1. Можно брать не только расширения, но и web-сервисы/сайты для портирования
2. Лучше всего: польза для работы, помощь в заработке, экономия времени
3. Оптимальный диапазон: от 10K пользователей (не миллионы от брендов)
4. Продукты с подпиской зарабатывают в 10x больше
5. Open source = в 100x проще разработка
