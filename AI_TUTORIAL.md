## Шаг 1: Загрузка контекста урока

### 1.1 Прочитай описание задачи

```
Файлы для чтения:
- CLAUDE.md                           # Общий контекст проекта
- sources/whisper-transcription/WEBINAR_INSIGHTS.md  # Инсайты вебинара
- prompts/IDEA_EVALUATION_TEMPLATE.md # Шаблон оценки
```

### 1.2 Требования модуля 1 (что нужно сделать):

| # | Задача | Результат |
|---|--------|-----------|
| 1 | Выбрать 10+ идей | Таблица с идеями |
| 2 | Оценить каждую по 8 критериям | Баллы 0-10 |
| 3 | Выявить RED FLAGS | Исключить плохие идеи |
| 4 | Забронировать 1 идею | Финальный выбор |

### 1.3 Критерии оценки (запомни):

```
1. Пользователи (0-10)      - от 10K, не миллионы брендов
2. Заработок (0-10)         - подписка? цена? доход?
3. Одна функция (ДА/НЕТ)    - RED FLAG если НЕТ
4. Простота (0-10)          - есть open source? GitHub?
5. Поиск ключа (0-10)       - Volume в Semrush
6. Софтовость (ДА/НЕТ)      - RED FLAG если <50% софта
7. Ключ свободен (ДА/НЕТ)   - RED FLAG если занят
8. KD Semrush (0-10)        - зелёная=10, оранж=7, красная=6
```

## Шаг 3: Генерация и анализ данных

### 3.1 SEO аналитика (Semrush)

Для каждой идеи собери:

```python
semrush_data = {
    "keyword": "css checker",          # Ключевое слово
    "volume_us": 1200,                  # Запросов/мес (США)
    "volume_global": 8500,              # Глобально
    "kd": 35,                           # Keyword Difficulty (%)
    "kd_zone": "оранжевая",             # зелёная/жёлтая/оранжевая/красная/алая
    "intent": "commercial",             # Намерение
    "serp_features": ["apps", "site"]   # Что в выдаче
}
```

### 3.2 AI анализ конкурентов

Для каждого конкурента проверь:

```python
competitor_analysis = {
    "name": "CSS Scan",
    "users": 10000,
    "has_subscription": True,
    "subscription_price": 88,
    "description_length": 2500,        # символов
    "languages_count": 15,             # локализаций
    "is_optimized": False,             # все 3 критерия?
    "one_function": True,
    "github_exists": True,
    "github_url": "https://github.com/miled/cssviewer"
}
```

### 3.3 Кэширование промежуточных результатов

Сохраняй в файлы:

```
reports/
├── ideas_raw.md           # Сырой список 10+ идей
├── semrush_cache.md       # Данные Semrush
├── competitors_cache.md   # Анализ конкурентов
└── FINAL_REPORT.md        # Финальный отчёт
```

**Формат кэша:**
```markdown
## Идея: CSS Checker

### Semrush данные (2025-12-04)
- Keyword: css checker
- Volume US: 1,200
- KD: 35% (оранжевая)

### Конкуренты
1. CSS Scan - 10K users, $88
2. CSS Viewer - 5K users, free
...
```

### 4.2 Формулы для расчётов

```python
# Потенциальная выручка
def calc_revenue(users, price=3):
    monthly = (users / 100) * price
    sale_price = users / 10
    return {"monthly": monthly, "sale": sale_price}

# Глобальный трафик
def calc_global_traffic(us_traffic):
    english = us_traffic * 10
    all_languages = english * 10
    return {"english": english, "all": all_languages}

# Итоговый балл
def calc_score(users, revenue, simplicity, keyword, kd):
    return users + revenue + simplicity + keyword + kd  # max 50
```

### 4.3 Правила оценки

**Баллы за пользователей:**
| Пользователей | Баллы |
|---------------|-------|
| <1K | 1-2 |
| 1K-10K | 3-5 |
| 10K-100K | 6-8 |
| 100K-1M | 9 |
| >1M (не бренд) | 10 |

**Баллы за KD (Semrush):**
| Зона | KD% | Баллы |
|------|-----|-------|
| Зелёная | 0-14 | 10 |
| Жёлтая | 15-29 | 10 |
| Оранжевая | 30-49 | 7-8 |
| Красная | 50-69 | 6 |
