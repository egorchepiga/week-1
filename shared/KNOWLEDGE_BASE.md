# ЧАСТЬ 1: ВЫБОР ИДЕИ

## 1.1 Критерии идеального продукта

### Нишевый vs Широкий

| Параметр | Нишевый | Широкий |
|----------|---------|---------|
| Связь с работой | ДА | НЕТ |
| Мин. пользователей | 10,000 | 100,000 |
| Мин. запросов US/мес | 500 | 2,000 |
| Конкуренция | Низкая | Высокая |
| Конверсия в оплату | Выше | Ниже |

**Вывод:** Начинать с НИШЕВОГО продукта!

### Источники идей

1. **Chrome Web Store** — chromewebstore.google.com
2. **App-Database.com** — база 200K+ расширений
3. **Похожие расширения** — раздел "Related" в CWS
4. **Сайты-сервисы** — можно упаковать в расширение
5. **Google выдача** — смотреть что не занято расширениями

### Запрещённые направления

| Тип | Примеры | Причина |
|-----|---------|---------|
| Бренды-монстры | Cisco, Grammarly, Adobe | Набрали за счёт бренда |
| Запрещённые бренды | Meta, LinkedIn, Claude | Юридические риски |
| Пиратство | YouTube Downloader | Блокировка |
| Несофтовые ниши | Кофейни, знаменитости | Не работает SEO |

## 1.2 Формулы расчёта

```python
# ВЫРУЧКА
monthly_revenue = (users / 100) * price    # 1% конверсия базовая
optimized_revenue = (users / 33) * price   # 3% после оптимизации
record_revenue = (users / 10) * price      # 10% рекорд буткемпа

# ПРОДАЖА ПРОЕКТА
sale_price = users / 10
# Пример: 100K users = $10,000

# ГЛОБАЛЬНЫЙ ТРАФИК
global_english = us_traffic * 10           # англоязычный мир
all_languages = global_english * 10        # все языки
# Пример: 500 US → 5,000 EN → 50,000 ALL

# ROI
roi_per_extension = users * 0.01 * price * 12  # годовая выручка
```

|---------|-----------|-------------------------|

| Оптимизированная | 3-5% | $9,000-15,000/мес |

# ЧАСТЬ 2: KEYWORD RESEARCH

## 2.1 Работа с Semrush

### Ключевые метрики

| Метрика | Что значит | На что смотреть |
|---------|------------|-----------------|
| Volume (без хвостов) | Точный запрос | Верхняя цифра |
| Volume (с хвостами) | С вариациями | Нижняя цифра |
| KD | Сложность | Цвет зоны |
| Intent | Намерение | Софтовость |
| SERP | Выдача | Есть ли расширения |

### Формула расчёта глобального трафика

```
US без хвостов: 2,400
Мир без хвостов: 42,600
Пропорция: 2,400/42,600 = 5.6%

US с хвостами: 103,500
Мир с хвостами: 103,500 / 0.056 = 1,848,214

+ Другие языки (×10): ~18,000,000 потенциальный охват
```

### Стратегия подбора ключей

1. **Основной ключ** — точное название функции
2. **Синонимы** — ChatGPT "Find 10 synonyms to: [ключ]"
3. **Long-tail** — Keyword Magic Tool → By Volume
4. **Related** — похожие запросы
5. **Нишевание** — добавление контекста (for Gmail, for LinkedIn)

## 2.2 Проверка конкурентов

### Признаки ОПТИМИЗИРОВАННОГО конкурента

| Критерий | Значение | Где смотреть |
|----------|----------|--------------|
| Название | = точный запрос | CWS |
| Description | > 3,000 символов | CWS |
| Локализация | > 30 языков | CWS |

**Если ВСЕ 3 выполнены → ниша ЗАНЯТА**
**Если хотя бы 1 НЕ выполнен → можно брать**

### Способы проверки конкурентов

1. **Google:** `[ключ] chrome` → смотрим расширения
2. **Semrush:** SERP фид внизу страницы
3. **App-Database:** Фильтры по description, languages

### Стратегии обхода конкурентов

```
Конкурент: "website blocker"
├── Вариант 1: "site blocker" (синоним)
├── Вариант 2: "web site blocker" (расширение)
├── Вариант 3: "focus mode" (другой угол)
└── Вариант 4: "website blocker for work" (нишевание)
```

## 2.3 Проверка софтовости

### Что считается СОФТОМ (хорошо)

- Сайты-сервисы
- Мобильные приложения
- Плагины, расширения
- Подборки приложений
- Любой софт

### Что НЕ является софтом (плохо)

- Сайты-статейники
- Wikipedia
- Маркетплейсы (Amazon, eBay)
- Каталоги, объявления
- Услуги

**Правило: > 50% софта в топ-10 = можно брать**

# ЧАСТЬ 5: ИНСАЙТЫ ИЗ ВЕБИНАРА

## 5.1 Скрытые стратегии

### "Монетка" — выбор при равенстве

> Если 2+ идеи набрали одинаковые баллы и не нарушают правила — выбирай случайно. Бизнес = награда за риск.

### "Двойной тест"

```
Гипотеза A: нишевый запрос, низкий KD
Гипотеза B: широкий запрос, высокий KD

Стратегия:
1. Запускаем A первым
2. Если A не взлетел → B тоже не взлетит (экономим время)
3. Если A взлетел → запускаем B для масштабирования
```

### "500 → 50,000"

```
500 запросов US кажется мало?

500 × 10 (англ. мир) × 10 (все языки) = 50,000

А если 2,000 US? → 200,000 потенциал
```

## 5.2 Ключевые цитаты

> "Польза – это экономия времени. Не сколько вы потратили, а сколько сэкономили человеку."

> "Десктоп становится устройством профессионала. Обыватели пользуются мобилкой."

> "Бездумное копирование пессимизируется алгоритмами."

> "Google очень любит свои расширения."

## 5.3 Статистика рынка

```
Всего расширений CWS: ~200,000
├── 0 пользователей:    65,000 (33%)
├── < 100 пользователей: 75,000 (38%)
├── < 1,000:            47,000 (24%)
├── > 10,000:            7,000 (3.5%) ← ЦЕЛЬ
└── > 100,000:           1,500 (0.75%)

KPI буткемпа: >10,000 users за год
Достигают: ~50% участников (vs 3.5% рынка)
```

# ЧАСТЬ 6: ПАТТЕРНЫ И ТАКТИКИ

## 6.1 Паттерн "Козырь расширений"

Расширения имеют преимущество в Google выдаче:
- Пробиваются даже при KD 90%+
- Вытесняют сайты с тем же названием
- Но НЕ вытесняют оптимизированные расширения

## 6.2 Паттерн "Шумный запрос"

```
Шумный запрос = много нерелевантных результатов

"bing chatgpt" → шумный
├── Только основной ключ работает
└── Хвосты не дадут трафика

Стратегия: брать только если основной ключ даёт достаточно трафика
```

## 6.3 Паттерн "Упаковка сайта"

```
Любой веб-сервис → расширение

1. Клик на иконку
2. Открывается ваш сайт
3. Пользователь работает

Быстрый способ получить расширение если есть сайт!
```

## 6.4 Паттерн "Экосистема"

```
Этап 1: Chrome Extension (основа)
         ↓
Этап 2: Google Workspace Marketplace (те же алгоритмы)
         ↓
Этап 3: Мобильное приложение (×40 конкуренции)
         ↓
Этап 4: Telegram бот (для переливания трафика)
         ↓
Этап 5: GPT Store (параллельно)
```
