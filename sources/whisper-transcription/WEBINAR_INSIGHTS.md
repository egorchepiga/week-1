## 1. ВЫБОР ИДЕИ: Нишевый vs Широкий проект

### Критерии нишевости (8-10 баллов):

| Фактор | Нишевый | Широкий |
|--------|---------|---------|
| Связь с работой | Да (профессия/учеба) | Нет |
| Тематичность | Одна аудитория | Разные группы |
| Минимум пользователей | 10,000 | 100,000 |
| Минимум запросов US/мес | 500 | 2,000 |

### Почему нишевые лучше:
- Выше платежеспособность (работа = деньги)
- Ниже конкуренция
- Проще реализация
- Выше конверсия в оплату

## 2. ФОРМУЛЫ РАСЧЁТА

```python
# Примерная месячная выручка
monthly_revenue = (users / 100) * subscription_price
# Пример: 10,000 юзеров × $3 = $300/мес

# Стоимость проекта при продаже (без выручки)
sale_price = users / 10
# Пример: 100,000 юзеров = $10,000

# Глобальный трафик из US-запросов
global_english = us_traffic * 10
all_languages = global_english * 10
# Пример: 1,000 US → 10,000 англ. → 100,000 все языки

# Конверсия в оплату
base_conversion = 0.01  # 1% базовая
optimized = 0.03 - 0.05  # 3-5% после оптимизации
record = 0.10  # 10% рекорд буткемпа
```

## 3. SEMRUSH: Ключевые метрики

### Что проверяем:
1. **Volume** - количество запросов в месяц (США)
2. **KD (Keyword Difficulty)** - сложность ранжирования
3. **Intent** - намерение пользователя
4. **SERP Features** - что показывается в выдаче

### Правила:
- Смотрим ТОЛЬКО точное вхождение (верхняя цифра)
- НЕ суммируем с вариациями запроса
- Проверяем софтовость выдачи (есть ли расширения в топе)
- KD до 30% - можно брать

## 7. ПОЧЕМУ CHROME WEB STORE

### Преимущества:
- Доверие Google к своему стору
- Ранжирование в поиске Google
- Меньше конкурентов чем в мобайле (в 40 раз)
- Более платежеспособная аудитория (профессионалы)

### Рынок расширений:
```
Всего расширений: ~200,000
0 пользователей: 65,000 (33%)
< 100 пользователей: 75,000 (38%)
< 1,000: 47,000 (24%)
> 10,000: 7,000 (3.5%) ← наш KPI
```

### Другие сторы:
- Safari, Firefox - нет своего поисковика, 10% рынка
- Edge Store - микро-процент
- Google Workspace Marketplace - те же алгоритмы

## 8. МОБИЛЬНЫЕ ПРИЛОЖЕНИЯ

### SEO вместо ASO:
- Не App Store Optimization
- А Search Engine Optimization
- Мобильные приложения тоже ищут в Google
- Меньше конкуренции чем внутри сторов

### Особенности:

- Строже модерация
- Делаем ПОСЛЕ успешного расширения

## 9. МОНЕТИЗАЦИЯ

### Ключевой принцип:
> Выручка = производная от ПОЛЬЗЫ
> Фокус на пользу, а не на деньги

### Что такое польза:
- Экономия времени пользователя
- НЕ количество времени разработчика
- Даже простая функция может давать пользу

### Статистика проектов:
- Ни один белый проект не закрылся полностью
- Бывают колебания ±20-50%
- Продолжительность жизни: неограниченна (если по правилам)
