# Этап 6: Оценка Keyword Difficulty (KD)

## Цель этапа
Понять, насколько сложно будет ранжироваться по выбранному ключевому слову.

## Что такое Keyword Difficulty

```
KD (Keyword Difficulty) — метрика Semrush (0-100%)
Показывает, насколько сложно попасть в топ выдачи по запросу.

ВАЖНО ДЛЯ РАСШИРЕНИЙ:
  KD рассчитан для САЙТОВ, а не расширений!
  Расширения имеют преимущество — Google продвигает их выше.
```

## Особенности KD для Chrome-расширений

### Расширения могут пробиваться при высоком KD
```
ПРИМЕР: Запрос "Bing ChatGPT"
  - KD = 91% (очень сложный для сайтов)
  - Но расширение заняло 2-е место после Bing!

ПРИЧИНА: Google любит свои расширения и автоматически продвигает их.
```

### Когда KD НЕ критичен
```
✓ Запрос явно "софтовый" (связан с программами)
✓ Запрос содержит "chrome", "extension", "addon"
✓ В выдаче уже есть расширения
✓ Расширение полностью оптимизировано
```

### Когда KD критичен
```
✗ Запрос общий (не связан с софтом)
✗ Выдача забита крупными сайтами
✗ Нет расширений в первых 10 результатах
```

## Алгоритм действий

### Шаг 6.1: Проверка KD в Semrush
```
ДЕЙСТВИЕ: Keyword Overview → ввести запрос → смотреть KD%

ИНТЕРПРЕТАЦИЯ:
  0-30%   — Легко (зелёная зона)
  31-60%  — Средне (жёлтая зона)
  61-80%  — Сложно (оранжевая зона)
  81-100% — Очень сложно (красная зона)
```

### Шаг 6.2: Принятие решения

```
ДИЛЕММА:
  Популярный запрос + высокий KD
  VS
  Нишевый запрос + низкий KD

РЕКОМЕНДАЦИЯ ДЛЯ НОВИЧКОВ:
  → Выбирать средний вариант (жёлтая/оранжевая зона)
  → Нишевый запрос с хорошим трафиком

РЕКОМЕНДАЦИЯ ДЛЯ ОПЫТНЫХ:
  → Можно брать высокий KD, если запрос софтовый
  → Расширение пробьётся благодаря оптимизации
```

### Шаг 6.3: Расчёт реального потенциала

```
ВАЖНО: 500 запросов в US — это не мало!

МУЛЬТИПЛИКАТОРЫ:
  1. US → Весь мир (английский) = ×10
  2. Английский → Все языки = ×10

ИТОГО: 500 US запросов ≈ 50,000 потенциальных запросов в мире

ПРИМЕР:
  500 запросов/мес (US, без хвостов)
  × 10 (весь англоязычный мир)
  × 10 (все языки с локализацией)
  = 50,000 потенциальных запросов
```

## Стратегии для разного KD

### Низкий KD (0-30%) — Зелёная зона
```
СТРАТЕГИЯ: Быстрый захват

ДЕЙСТВИЯ:
  ✓ Запускать сразу
  ✓ Базовая оптимизация достаточна
  ✓ Ожидать результат через 2-4 недели

РИСК: Возможно, трафика мало (проверить объём!)
```

### Средний KD (31-60%) — Жёлтая зона
```
СТРАТЕГИЯ: Сбалансированный подход

ДЕЙСТВИЯ:
  ✓ Полная оптимизация описания
  ✓ 30+ переводов
  ✓ Качественные скриншоты
  ✓ Ожидать результат через 1-2 месяца

ОПТИМАЛЬНО: Лучший баланс трафик/конкуренция
```

### Высокий KD (61-100%) — Красная зона
```
СТРАТЕГИЯ: Максимальная оптимизация

ДЕЙСТВИЯ:
  ✓ Идеальное совпадение названия с запросом
  ✓ Описание 3000+ символов
  ✓ 50+ переводов
  ✓ Регулярные обновления
  ✓ Работа с отзывами
  ✓ Ожидать результат через 2-4 месяца

ВАЖНО: Работает только для софтовых запросов!
```
