# Каталог ошибок при выборе идей для Chrome Extension

> **Источник:** Транскрипции разборов Week 1 (ЧТ 31.10, ПТ 01.11, СБ 02.11.2024)
> **Для AI:** Используй этот каталог для проверки идей перед валидацией

---

## Критические ошибки (RED FLAGS)

### 1. Отсутствие конкурентов
**Описание:** Нет ни одного работающего продукта на эту тему (ни extension, ни сайт, ни app)
**Почему плохо:** 8 миллиардов людей, вероятность быть первым минимальна
**Пример:** Convert Numbers to Excel — нет extension, но есть сайты-конверторы
**Решение:** Искать конкурентов среди сайтов, мобильных приложений, не только extension

### 2. Не софтовая выдача
**Описание:** В SERP по запросу показываются статьи, Wikipedia, форумы вместо софта
**Почему плохо:** Google считает запрос информационным, не коммерческим
**Как проверить:** Ввести запрос в Semrush SERP и посмотреть топ-10
**Решение:** Добавить окончание -ER (testing → tester) часто делает запрос софтовым

### 3. Слово FREE в названии
**Описание:** Добавление слова "free" в название расширения
**Почему плохо:** Модерация Chrome не пропустит или удалит на дистанции
**Решение:** Никогда не добавлять FREE в название

### 4. Запрещённые бренды
**НЕТ:** Meta, LinkedIn, Claude, Gemini, Instagram, Facebook
**ДА:** YouTube, Amazon, Google Sheets, ChatGPT, OpenAI
**Причина:** Meta/LinkedIn активно блокируют расширения для своих платформ

### 5. Продукты на втором манифесте (MV2)
**Описание:** Базируетесь на open-source с Manifest V2
**Почему плохо:** Chrome удаляет MV2 расширения, они скоро перестанут работать
**Как проверить:** Смотреть manifest.json в репозитории
**Решение:** Искать только MV3 или переписывать с нуля

## Ошибки оценки запроса

### 6. Доверие интенту Semrush
**Описание:** Полагаться на показатель Intent (Commercial/Informational) в Semrush
**Почему плохо:** Semrush показывает случайные значения, не коррелирует с реальностью
**Пример:** MP3 to WAV и MP3 to WAV без точек — разные Intent при идентичности
**Решение:** Проверять софтовость вручную через SERP

### 7. Cost Per Click = 0 как red flag
**Описание:** Отказ от запроса из-за нулевого CPC
**Почему неверно:** CPC показывает только наличие рекламы, не покупательскую способность
**Решение:** CPC — дополнительный фактор, не решающий

### 8. Малый Volume = плохой запрос
**Описание:** Отказ от запроса с Volume < 500
**Нюанс:** Формула Global = US × 10 × 10 (для EN + для других языков)
**Пример:** 720 поисков → 7,200 Global EN → 72,000 со всеми языками
**Решение:** Минимум 500, но учитывать множитель ×100 для глобального трафика

### 9. Игнорирование зелёного KD
**Описание:** Радость от низкого KD без проверки ботоводности
**Почему опасно:** Зелёная зона (< 30 KD) часто накручена ботами
**Как проверить:** Открыть тренд запроса в Semrush — должен быть стабильный, не "пик-ноль-пик"
**Решение:** Всегда проверять тренд для запросов с низким KD

## Ошибки в оптимизации

### 10. Слишком длинное название (> 30 символов)
**Описание:** Название больше 30 символов с лишними словами
**Пример:** 73 символа в названии — продукт не растёт
**Решение:** Название = Запрос, ничего лишнего

### 11. Добавление слов не из запроса
**Описание:** Добавление Google, Chrome, Extension если их нет в запросе
**Почему плохо:** Расфокусирует поисковый алгоритм
**Правило:** В название включать ТОЛЬКО то, что есть в запросе

### 12. Убрать артикли или союзы
**Описание:** Удаление a/the/for из запроса при создании названия
**Почему плохо:** Это часть запроса, меняет SEO-характеристики
**Решение:** Если артикль есть в запросе — сохранять

### 13. Недостаточное изменение названия
**Описание:** Думать что MP3 и MP.3 — разные названия
**НЕ считается изменением:**
- Регистр букв (mp3 vs MP3)
- Точки (MP3 vs M.P.3)
- Окончание S (множественное число)
- Окончание -ER (convert vs converter)
**Считается изменением:**
- Добавить слово
- Убрать слово
- Заменить слово

### 14. Мало языков в локализации
**Описание:** Только английский или < 10 языков
**Почему плохо:** Теряете ×10 трафика от других языков
**Стандарт:** Минимум 30+ языков в описании

### 15. Короткое описание
**Описание:** Long Description < 3000 символов
**Почему плохо:** Меньше ключевых слов для индексации
**Стандарт:** Минимум 3000 символов

## Ошибки выбора ниши

### 16. Не для работы
**Описание:** Продукт для развлечения, а не для работы
**Почему плохо:** Desktop = инструмент профессионалов, обыватели на телефонах
**Примеры плохих ниш:** Религиозные приложения, игры, развлечения
**Хорошие ниши:** AI, PDF, конвертеры, dev tools, дизайнерские инструменты

### 17. Image Downloader ниша
**Описание:** Скачивание картинок с сайтов
**Проблемы:**
- Аудитория: СНГ, Азия, Африка (низкая конверсия)
- Околопиратская тема (риски с эквайрингом)
- Только реклама как монетизация
**Решение:** Если хочется — только реклама, но лучше другая ниша

### 18. Донаты как модель монетизации
**Описание:** Рассчитывать на пожертвования пользователей
**Почему не работает:** Разница с подписками в 2-3 порядка
**Пример:** 6М пользователей VidIQ зарабатывает меньше чем мидл-зарплата
**Решение:** Только подписки, one-time payment, реклама

### 19. Ошибка выжившего
**Описание:** Копировать неоптимизированный продукт с пользователями
**Почему опасно:** < 3% таких продуктов успешны случайно
**Пример:** 30K пользователей без оптимизации — исключение, не правило
**Решение:** Следовать методике оптимизации независимо от примеров

## Ошибки реализации

### 20. Начинать со сложного
**Описание:** Первый продукт — AI генератор, сложная логика
**Почему плохо:** Долго, дорого, не понимаешь работает ли метод
**Примеры:**
- YouTube Tag Generator (сложно) → лучше начать с проще
- Grammar Checker (сложно) → лучше Character Counter
**Решение:** Начать с простого, потом усложнять

### 21. Встраивание через сайт
**Описание:** Сначала сделать сайт, потом extension
**Почему опасно:** Модерация не пропустит без нюансов
**Решение:** Extension = сайт встроенный, но есть специфика (неделя 2)

### 22. Отсутствие open-source
**Описание:** Не искать готовые решения на GitHub
**Проблема для не-разработчиков:** Сложно реализовать с нуля
**Что искать в репозитории:**
- Ссылка на рабочий extension
- Manifest V3
- Активные коммиты
**Решение:** Если нет open-source и не разработчик — сменить идею

## Ошибки валидации

### 23. Проверять только Chrome Web Store
**Описание:** Искать конкурентов только среди extension
**Почему неполно:** Конкуренты могут быть на сайтах, в мобильных приложениях
**Пример:** Convert Numbers to Excel — нет extension, но есть CloudConvert, Zamzar
**Решение:** Semrush Traffic Analytics для сайтов-конкурентов

### 24. Не проверять подписки конкурентов
**Описание:** Видеть extension и не открывать их pricing
**Почему плохо:** Не понимаете потенциал монетизации
**Как проверять:**
- Открыть топ-3 сайта из SERP
- Найти Pricing страницу
- Убедиться что платные планы есть
**Нюанс:** Подписки могут быть только для определённых локалей

### 25. Игнорировать Tier-1 страны
**Описание:** Не смотреть географию трафика
**Tier-1 (платят):** USA, UK, Canada, Germany, Australia, Japan, Korea
**Tier-2 (меньше):** Остальная Европа
**Tier-3 (слабо):** СНГ, Азия, Африка, Латинская Америка
**Решение:** Смотреть в Semrush откуда трафик у конкурентов

## Чек-лист проверки идеи (для AI)

```
□ Есть конкуренты (extension ИЛИ сайты ИЛИ apps)?
□ SERP софтовый (топ-10 = софт, не статьи)?
□ Volume >= 500 (US)?
□ KD <= 70 (оранжевая зона ок)?
□ Тренд стабильный (не бото-накрутка)?
□ Есть платные конкуренты (pricing page)?
□ Нет MV2 зависимости?
□ Нет запрещённых брендов?
□ Для работы (не развлечение)?
□ Tier-1 география?
```
