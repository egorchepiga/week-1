## Ключевые правила (из разборов)

### Выбор запроса
1. **Начинать с узкого** — проще ранжироваться, потом расширять
2. **-ER делает софтовым** — testing → tester
3. **Слова Chrome/Google/Extension** — добавлять ТОЛЬКО если есть в запросе
4. **Минимальное изменение** — одно слово достаточно для нового продукта

### Проверка ниши
1. **Конкуренты обязательны** — если нет, идею не брать
2. **Не только extension** — искать сайты, мобильные apps
3. **Semrush Traffic Analytics** — проверять трафик сайтов-конкурентов
4. **Tier-1 страны** — USA, UK, Germany, Australia, Japan

### Монетизация
1. **Подписки лучше донатов** — разница в 100-1000x
2. **Pricing page** — всегда проверять у конкурентов
3. **Локаль-специфичные подписки** — могут не показываться в вашей стране
4. **Реклама** — запасной вариант для серых ниш

### Оптимизация
1. **Название = Запрос** — ничего лишнего
2. **30+ языков** — обязательно
3. **3000+ символов описания** — обязательно
4. **MV3 только** — MV2 удаляется

## Стратегии роста

### Экосистема продуктов
```
1. Запустить 3-5 extension на близкую тему
2. Собрать трафик на каждом
3. Создать объединяющий сайт
4. Сайт продвигается от ссылок extension
5. Сайт сливает трафик на лучший extension
6. Extension монетизирует (LTV выше чем у сайта)
```

### Вторая попытка
- Если не зашло — изменить название (добавить/убрать слово)
- MP3 to Excel → MP3 to XLS — разная динамика
- Много примеров успеха со второй попытки

### Google Workspace Marketplace
- Альтернатива Chrome Web Store
- Продукты для Sheets, Docs, Slides
- Та же механика продвижения
- Пример: ChatGPT for Sheets — 6M пользователей
